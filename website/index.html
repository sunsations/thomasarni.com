Blogging

about software and the web.

One of the greatest ways on how remote people can interact and exchange ideas with each other is blogging. Never in human history, has it been easier to get in contact with like-minded people in the world.

This is a chance I don't want to miss out, because I'm an avid reader of various blogs and learned a lot of it. For java enthusiast like me there are great blogs like Java Code Geeks or Resources for Java developers by IBM developerWorks and many more.

A full list of Java and software engineering related blogs I read, can be found in my Google Bundle.

My favorite RSS feed readers are Google Reader and Byline on my iPhone.

Software Engineering

and it's best practices.

I'm a Java software engineer with a strong information retrieval background. Currently my main interest is on web development.

Agile

I believe that agile methodologies (like Scrum) are currently the most effective way to develop great software. I hope that more companies will start using Scrum and that the agile approach is backed up by the management.

Education

I am a strong believer that education at all levels is one of the most important necessities for success. The truly educated men never graduate. Zühlke, my employer, offers four week per year for personal education. I simply love this and hope more companies will follow in the future.

Especially in the software industry, where things used to change so rapidly, it is crucial to keep up to date and learn new things.

Books

Books that every Java software engineer should have read (and more importantly understood) are in my opinion: Effective Java by Joshua Bloch, Refactoring by Martin Fowler, Clean Code by Uncle Bob or Design Patterns by Erich Gamma to name a few. They represent compressed knowledge of the masters in the field of software engineering.

Principles

To know the basic principles like KISS, SOLID, DRY or YAGNI is off pat.
